CWM-E API (1.0.0)

Download OpenAPI specification:

Canonical CWM-E OpenAPI contract for Project LOGOS. See Project LOGOS spec: Section 3.3 (CWM-E - Emotional/Social Layer).

CWM-E (Causal World Model - Emotional/Social) is the third layer of Sophia's world model, complementing:

  • CWM-A: Abstract/commonsense reasoning in the HCG
  • CWM-G: Grounded physics/perception (JEPA-style predictions)
  • CWM-E: Affective/social signals derived from reflection

CWM-E reflects on stored memories (persona entries) to infer confidence, trust, or other affective signals. These emotion states are written back to the HCG as tags on processes and entities, allowing planners and Apollo to adjust behavior and tone.

Create Emotion State

Create a new EmotionState node in Neo4j representing an affective signal. Emotion states can be tagged on processes and entities to influence planning and interaction behavior.

Request Body schema: application/json
required
emotion_type
required
string
Enum: "confident" "cautious" "curious" "neutral" "concerned"

Type of emotion (e.g., confident, cautious, curious)

intensity
required
number <float> [ 0 .. 1 ]

Confidence/strength of emotion (0.0 to 1.0)

context
string

Optional description of what triggered this emotion

Responses

Local development server

http://localhost:8000/cwm-e/emotions

Run Reflection Job

Trigger a CWM-E reflection job that analyzes recent PersonaEntry nodes and generates EmotionState nodes based on sentiment analysis.

This is the core CWM-E inference mechanism that derives affective signals from stored memories and writes them back to the HCG.

query Parameters
limit
integer [ 1 .. 100 ]
Default: 10
Example: limit=10

Number of recent persona entries to analyze

Responses

Local development server

http://localhost:8000/cwm-e/reflect

Response samples

Content type

  • "emotions": [

      • "emotion_type": "confident",
